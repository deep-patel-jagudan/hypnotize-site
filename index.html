01

Delta-time lerp

A fixed 0.05 factor runs 2.4× too fast at 144fps. The fix: 1 - pow(smooth, δt × 60) — the same visual speed at any frame rate, always.

02

Adaptive tiers

GPU benchmark on first paint detects HIGH / MID / LOW. Particles and blur scale down gracefully on budget phones. The vibe stays intact.

03

Touch & gyro

Mobile gets gyroscope parallax via DeviceOrientation. Falls back to touchmove. Both normalize to ±1 and feed the exact same lerp pipeline as mouse.

04

BPM-sync

When the track plays, the Web Audio API detects the beat and pulses the ring at exactly the right interval. Motion and music breathe together.

02 — The formula

One formula.
Every screen.

JS — frame-rate independent lerp 
// ❌ WRONG — fixed lerp breaks at 144fps
value += (target - value) * 0.05;

// ✅ CORRECT — identical at any frame rate
const delta  = Math.min((now - lastTime) / 1000, 0.1);
const factor = 1 - Math.pow(0.92, delta * 60);
value += (target - value) * factor;

// Tuning the constant (0.92):
//  0.70 → snappy   0.85 → balanced
//  0.92 → butter   0.97 → very slow


JS — mobile gyro + touch fallback 
if (mobile) {
  // Gyroscope (most immersive on phone)
  window.addEventListener('deviceorientation', e => {
    rawX = Math.max(-1, Math.min(1, e.gamma / 25));
    rawY = Math.max(-1, Math.min(1, e.beta  / 25));
  });
} else {
  // Desktop mouse
  window.addEventListener('mousemove', e => {
    rawX = (e.clientX / innerWidth  - 0.5) * 2;
    rawY = (e.clientY / innerHeight - 0.5) * 2;
  });
}

03 — Beat sync

Motion that breathes
with the music.

Hypnotize runs at ~95 BPM. The ring you see on screen pulses at exactly that interval — 632ms per beat. The Web Audio API feeds real frequency data to the EQ bars, so the player bounces to the actual bass and treble of whatever track you load. Change the BPM constant in the JS to match your song.
